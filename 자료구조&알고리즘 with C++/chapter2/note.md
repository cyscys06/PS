### 비선형 문제
- 선형 자료구조로 표현 불가능한 문제들
- 예시: 계층적 문제, 순환 종속성 문제
- 데이터가 선형(일직선)으로 저장되지 않고, 하나의 데이터에서 여러 데이터들 중 하나로 접근할 수 있는 경우

### 계층적 문제
- 어떤 구성이 계층적인 구성을 이루는 경우
- 예시: 회사 조직도, 대학 교과과정 계층도

### 순환 종속성
- 예시: 친구 관계
- 하나의 노드에서 여러 다른 노드로 접근할 수 있는 경우
- 여러 갈래 형태로 갈라질 수 있으므로, 선형 형태가 아님. 따라서 선형 자료구조로 나타낼 수 없음

### 트리: 상하 반전된 형태
- 노드와 노드 사이를 연결하는 간선(edge)를 사용해서 트리를 구성할 수 있음(나무 형태)
- 루트 노드: 트리의 중심이 되는 노드이며, 트리 구조의 맨 위에 있는 노드를 말함
- 실제 나무와 달리 뿌리(루트 노드)가 맨 위에 있으므로 상하 반전된 형태임

### 트리 순회
1. 전위 순회
-> 1. 현재 노드를 방문
-> 2. 현재 노드의 왼쪽 하위 노드를 방문
-> 3. 현재 노드의 오른쪽 하위 노드를 방문
-> 4. 위 과정을 재귀적으로 반복
2. 중위 순회
-> 1. 현재 노드의 왼쪽 하위 노드를 방문
-> 2. 현재 노드를 방문
-> 3. 현재 노드의 오른쪽 하위 노드를 방문
-> 4. 위 과정을 재귀적으로 반복
3. 후위 순회
-> 1. 현재 노드의 왼쪽 하위 노드를 방문
-> 2. 현재 노드의 오른쪽 하위 노드를 방문
-> 3. 현재 노드를 방문
-> 4. 위 과정을 재귀적으로 반복
4. 레벨 순회
-> 트리의 루트 트리부터 시작
-> 트리 단계(레벨)별로 차례대로 방문


### 다양한 트리 구조

### 이진 검색 트리
- 왼쪽 자식 노드의 값 <= 부모 노드의 값 <= 오른쪽 자식 노드의 값 의 특성을 가짐
- 노드 검색: 
  1. 루트 노드 값과 찾고자 하는 노드 값을 비교
   1. 루트 노드 값이 더 작은 경우: 찾고자 하는 노드 값이 루트 노드의 왼쪽에 있음
   2. 루트 노드 값이 더 큰 경우: 찾고자 하는 노드 값이 루트 노드의 오른쪽에 있음
  2. 비교 결과에 따라 루트 노드의 왼쪽/오른쪽으로 이동
  3. 이동한 노드에서 1~2번 과정을 반복
- 노드 추가:
  1. 노드 검색 루트와 같이 이동
  2. 만약 현재 노드의 자식 노드(왼쪽이나 오른쪽)로 이동해야 하는데, 해당 위치에 노드가 없을 경우, 추가할 값을 가지는 새로운 노드를 추가
- 노드 삭제:
  1. 노드 검색 루트와 같이 이동하면서, 삭제할 값을 가진 노드가 있는지 탐색
  2. 삭제할 값을 가진 노드를 찾은 경우:
    1. 해당 노드가 자식 노드가 없는 경우
    -> 해당 노드만 삭제
    2. 해당 노드가 자식 노드를 하나만 가진 경우
    -> 해당 노드를 삭제 후, 해당 노드의 부모 노드가 해당 노드의 자식 노드를 가리키도록 수정
    3. 해당 노드가 자식 노드를 둘 다 가진 경우
    -> 현재 노드를 후속 노드로 대체
    후속 노드: 현재 노드 값보다 큰 값들 중 최솟값을 가진 노드
    후속 노드를 찾는 방법: 현재 노드의 오른쪽 서브 트리(현재 노드보다 큰 값을 가진 노드들로 구성된 트리)로 이동한 후, 
    해당 서브 트리에서 가장 왼쪽에 있는 노드(현재 노드 값보다 큰 값들 중 가장 최솟값을 가진 노드)로 이동
      1. 위 방식으로 삭제할 노드의 후속 노드를 탐색
      2. 현재 노드 값을 후속 노드 값으로 변경
      3. 후속 노드를 삭제
      -> 후속 노드는 개념상 오른쪽 자식 노드(후속 노드 값보다 큰 값들 중 하나)만 가질 수 있음. 
      만약 후속 노드에 왼쪽 노드(후속 노드 값보다 작은 값을 가진 노드)가 존재한다면, 
      현재 노드는 후속 노드의 기준을 만족하지 못하므로(본인보다 더 작은 값을 가진 노드가 존재하므로) 후속 노드가 아님
