### 문제 설명
- 해결해야 할 여러 개의 작업이 존재
- 각각의 작업은 작업 ID, 특정 시작 시간 및 종료 시간(즉 작업 시간)을 가짐
- 정해진 시간 안에 최대한 많은 작업을 완료하는 것이 목표
- 즉, 정해진 시간 및 여러 작업들의 시작시간/종료시간을 고려하여 진행한 작업 수의 최댓값을 구하는 것이 목표

### 기능 구현 목록
1. 작업 구조체
- [x] 작업 ID, 시작시간, 종료시간을 멤버 변수로 가짐
- [x] 정렬에 사용할 <연산자 오버로딩(종료시간을 기준으로 비교)
- [x] 작업에 대한 정보를 출력하기 위한 <<연산자 오버로딩
2. N개의 작업들을 저장할 작업 리스트 
- [x] 저장할 데이터 타입은 작업 구조체 형임
3. 스케쥴링된 작업들을 저장할 벡터(반환형이 벡터이므로)
- [x] 마찬가지로 저장할 데이터 타입은 작업 구조체 형임
4. 여러 작업들을 스케쥴링할 함수(내부에 여러 함수 가짐)
- [x] 종료시간을 기준으로 전체 작업 리스트를 정렬
- [x] 가장 빠른 종료시간을 가진 작업을 선택하여 스케쥴링된 작업들을 저장할 리스트에 저장
- [x] 현재 스케쥴링된 작업과 시간이 겹치는 다른 선택되지 않은 작업들(즉, 선택된 작업의 종료 시간보다 먼저 시작하는 작업들)을 삭제
- 위와 같은 과정을 리스트의 작업이 비어있을 때까지 반복, 반환형은 스케쥴링된 작업을 저장할 리스트
